Investigación sobre Markdown
¿Qué es Markdown y por qué se utiliza en proyectos de software?

Markdown es un lenguaje de marcado ligero que permite formatear texto de manera simple, usando solo caracteres comunes como #, *, -, [], etc.
Fue diseñado para que cualquier persona pueda escribir documentos fácilmente sin necesidad de usar editores complicados.

¿Por qué se usa en proyectos de software?

Para escribir documentación clara sin usar Word o herramientas pesadas.

Porque se puede leer incluso sin procesarlo (el archivo puro también es entendible).

Porque GitHub, GitLab y la mayoría de plataformas lo interpretan automáticamente.

Se usa en archivos importantes como README.md, CHANGELOG.md, CONTRIBUTING.md y más.

Ejemplo práctico de Markdown

Aquí tenemmos un ejemplo completo más usado:

1. Encabezados
# Título 1
## Título 2
### Título 3

2. Listas

Lista con viñetas

- Elemento 1
- Elemento 2
  - Sub-elemento


Lista numerada

1. Primer paso
2. Segundo paso
3. Tercer paso

3. Enlaces
[Ir a GitHub](https://github.com)

4. Tablas
| Producto | Precio | Cantidad |
|----------|--------|----------|
| Manzanas | $1.00  | 5        |
| Pan      | $0.50  | 2        |

5. Imágenes
![Descripción de la imagen](https://ruta-de-la-imagen.com/imagen.png)

Ventajas de usar Markdown con GitHub

GitHub renderiza automáticamente los archivos .md, haciéndolos fáciles de leer.

Permite crear documentación profesional sin usar Word.

Es perfecto para colaborar con otros porque se puede versionar (control de cambios).

Facilita la creación de:

README.md

Wikis
